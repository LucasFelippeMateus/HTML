A estrutura básica da programação em html consiste nos seguintes comandos:

    < html > < / html > para indicar que o que está sendo escrito deve ser entendido como html;
    < head > < / head > para indicar o cabeçalho da página;
    < body > < / body > para indicar o corpo da página;
Lembrando que os comandos são escritos dentro de tags <> e estas devem ser inicializadas: < comando > e finalizadas: < / comando >.

No cabeçalho da página serão inseridas tags de configuração do documento que não serão visíveis na página.

No corpo da página será incluído todo o conteúdo visível ou de interação com o usúario, por exemplo, imagens, vídeos, tabelas, formulários, dentre outros.

Dentro do cabeçalho ainda é possível utilizar outra tag conhecida como < title > < / title > para indicar o título da página que irá aparecer na aba do navegador.

Os navegadores não interpretam a tecla 'enter' inserida no html, então para fazer quebras de linhas é preciso utilizar a tag break < br / >. Veja o exemplo a seguir:

    Sem < br / >
    Com
    < br / >

Em relação aos textos, não é uma boa prática deixa-los soltos no html. É sempre importante inseri-los dentro de uma determinada tag para indicar se é um parágrafo, um título, um subtítulo e assim por diante. Além disso, vale ressaltar que em html não formatamos o texto diretamente mas sim as tags.

Uma das tags que utilizamos quando trabalhamos com textos são os headers que variam de h1 até h6 sendo que o 1 é o nível hierarquico mais alto. A seguir segue um exemplo de utilização de cada um dos headers:

    Título 1

    Título 2 (subtítulo de 1)

    Título 3 (subtítulo de 2)

    Título 4 (subtítulo de 3)

    Título 5 (subtítulo de 4)
    Título 6 (subtítulo de 5)

Além das tags para os títulos e subtítulos, nos textos trabalha-se bastante com a tag p para indicar diferentes parágrafos na página. A própia tag p já faz o gerenciamento de quebra de linhas quando nescessário.

Recomendasse não utilizar caracteres especiais, nem espaço e nem iniciar com letra maiúscula no nome do arquivo. Este deve ser salvo na extensão .html
